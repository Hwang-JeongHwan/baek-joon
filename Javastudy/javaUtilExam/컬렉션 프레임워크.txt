컬랙션 프레임 워크
java.util 패키지에는 자료를 다룰 수 있는 자료구조 클래스가 다수 존재함
자료구조 클래스들을 컬렉션 프레임워크 라고 한다.

자료구조란 자료를 저장할 수 있는 구조

책을 보관하기 위해 책장을 이용한다던지 이런 방법을 이용함 -> 다양한 자료들을
다양한 방식으로 관리하기 위한 방식이 필요함
=> 이러한 방법을 제공하는것을 자료구조, collection framwork라고함

Collection Framwork에서 가장기본이 되는 인터페이스는
Collection interface임
-> 컬랙션 인터페이스는 여기에 자료가 있다 라는 것을 표현함 
-> 중복도 허용하고 자료가 저장되는 순서는 기억하지 못하는 것이 컬랙션 인터페이스임
add -> 추가하기 위한 메소드
iterator -> 자료를 하나씩 꺼내기위한 메소드
size -> 사이즈를 구하기 위한 메소드
라는 메소드를 가지고있음
컬랙션은 저장된 순서를 기억하지 못하기 떄문에 첫번째 자료를 달라, 두번째 자료를 달라
이런식으로 할수없음

컬랙션은 저장된 자료를 하나씩 하나씩 꺼낼수 있는 이터레이터라는 인터페이스를 의존하고
반환함
iterator는 hasNext()라는 메소드를 가지고 있음
-> 꺼낼것이 있는지 없는지 살펴보는 메소드임
하나씩 자료를 꺼낼때 사용할수 있는next()라는 메소드를 가짐

Set이라는 자료구조는 중복을 허용하지 않는 자료구조를 표현한 인터페이스 이고 
Collection 인터페이스를 상속받음
Set인터페이스가 가지고 있는 add()메소드는 같은 자료가 있으면 false라는 값을 반환해주고
같은 자료가 없으면 true를 반환해주는 메소드를 가지고 있음

List라는 자료구조는 중복을 허용하면서 순서를 기억하는 자료구조임
Set인터페이스와 마찬가지로 Collection 인터페이스를 상속함
List 자료구조는 순서를 기억하므로 0번째 1번째 2번째 이런식으로 순차적으로
자료를 꺼내줄수 있는 get(int) 메소드가 있고 매개변수 int값이 인덱스 번호를 나타낼
수 있음

Map이라는 자료구조는 Key와 value를 가지는 자료구조이고 Set에 의존함
저장할 때에 put(Object, Object)메소드를 이용해서 키와 밸류를 함께 저장함
원하는 값을 꺼낼때는 키를 매개변수로 받아들이는 get(Object)메소드를 이용해서 값을 꺼냄
그러면 키에 해당하는 값을 한번에 꺼낼수 있음
맵의 저장되어 있는 모든 키들은 중복된 값을 가지면 안됨
키의 이러한 특징 때문에 Map 은자신이 가지고있는 모든 키들에 대한 정보를 읽어 올수 있는
Set을 반환하는 keySet()이라는 메소드를 가지고 있음

자바 유틸 패키지에는 워ㅣ에서 설명한 인터페이스를 구현하고 있는 클래스를
많이 가지고있음
